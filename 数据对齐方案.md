# 鹤问湖水质净化厂 - 数据对齐方案

- ✅ 菜单结构已调整为净水厂工艺流程

## 二、净水厂工艺流程分析

### 2.1 工艺流程模块
1. **预处理** - 粗格栅、细格栅、沉砂池、提升泵房
2. **I段AAO** - 厌氧池、缺氧池、好氧池（第一段）
3. **二沉池及出水** - 二次沉淀池、出水监测
4. **污泥脱水** - 污泥浓缩、脱水机房
5. **高效沉淀池** - 混凝、絮凝、沉淀
6. **反硝化深床滤池** - 深度过滤、反硝化
7. **加药系统** - PAC、PAM等药剂投加
8. **II段AAO** - 厌氧池、缺氧池、好氧池（第二段）
9. **鼓风机房** - 曝气系统、鼓风机组

### 2.2 关键监测指标

#### 水质指标
- COD（化学需氧量）mg/L
- BOD（生化需氧量）mg/L
- NH3-N（氨氮）mg/L
- TN（总氮）mg/L
- TP（总磷）mg/L
- SS（悬浮物）mg/L
- pH值
- 溶解氧 DO mg/L
- 浊度 NTU
- 温度 ℃


#### 流量指标
- 进水流量 m³/h
- 出水流量 m³/h
- 回流流量 m³/h
- 污泥流量 m³/h

#### 液位指标
- 各池体液位 m
- 储药罐液位 m

#### 设备运行指标
- 水泵运行状态、频率、电流、功率
- 鼓风机运行状态、频率、风量、压力
- 搅拌器运行状态、电流
- 加药泵运行状态、流量
- 脱水机运行状态、泥饼含水率

#### 能耗指标
- 总用电量 kWh
- 单位水量能耗 kWh/m³
- 各工艺段能耗

## 三、数据结构对齐方案

### 3.1 核心数据类型定义

需要创建新的类型定义文件：`src/api/types/water-treatment.ts`

```typescript
// 水质监测数据
export interface WaterQualityData {
  cod: number        // COD mg/L
  bod: number        // BOD mg/L
  nh3n: number       // 氨氮 mg/L
  tn: number         // 总氮 mg/L
  tp: number         // 总磷 mg/L
  ss: number         // 悬浮物 mg/L
  ph: number         // pH值
  do: number         // 溶解氧 mg/L
  turbidity: number  // 浊度 NTU
  temperature: number // 温度 ℃
}

// 流量数据
export interface FlowData {
  inflowRate: number      // 进水流量 m³/h
  outflowRate: number     // 出水流量 m³/h
  returnFlowRate: number  // 回流流量 m³/h
  sludgeFlowRate: number  // 污泥流量 m³/h
}
```

   - 数据采集异常

## 八、技术建议

### 8.1 数据生成策略
- 使用真实的水质指标范围
- 模拟昼夜变化规律（进水流量白天高、夜间低）
- 模拟工艺控制逻辑（如溶解氧控制在2-4mg/L）
- 添加合理的数据波动

### 8.2 性能优化
- 使用数据缓存减少重复计算
- 图表数据按需加载
- 大数据量使用虚拟滚动

### 8.3 可扩展性
- 预留多个污水厂的数据结构
- 支持动态配置监测点位
- 支持自定义告警规则
- 支持自定义报表模板

## 十、参考资料

### 污水处理工艺参数参考值
- 进水COD: 200-400 mg/L
- 出水COD: <50 mg/L (一级A标准)
- 进水氨氮: 25-45 mg/L
- 出水氨氮: <5 mg/L (一级A标准)
- 进水总氮: 30-50 mg/L
- 出水总氮: <15 mg/L (一级A标准)
- 进水总磷: 3-5 mg/L
- 出水总磷: <0.5 mg/L (一级A标准)
- 好氧池溶解氧: 2-4 mg/L
- 污泥浓度MLSS: 3000-5000 mg/L
- pH: 6.5-8.5
